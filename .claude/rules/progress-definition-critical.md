## 進捗100%の定義（絶対基準）

### 🔴 進捗100% = 1000万円の現金収入を得た瞬間

これが唯一の100%達成条件である。それ以外の状態は全て100%未満。

---

## 進捗率の計算式

```
進捗率(%) = (累計収益 / 1000万円) × 100

例:
- 収益 0円      → 進捗 0%
- 収益 10万円   → 進捗 1%
- 収益 100万円  → 進捗 10%
- 収益 500万円  → 進捗 50%
- 収益 1000万円 → 進捗 100%
```

## 禁止事項

### 🔴 絶対禁止

1. **進捗率の過大評価**
   - 「コード完成 = 80%」は虚偽
   - 「テスト通過 = 90%」は虚偽
   - 「ストア申請待ち = 95%」は虚偽

2. **楽観的な進捗報告**
   - 「ほぼ完成」→ 収益0円なら進捗0%
   - 「あとは公開するだけ」→ 収益0円なら進捗0%
   - 「順調に進んでいる」→ 収益0円なら順調ではない

3. **技術的達成感による錯覚**
   - ビルド成功の喜び → 収益には無関係
   - テスト100%通過 → 収益には無関係
   - 美しいUI完成 → 収益には無関係

## 批判的思考の義務

### 各AIが常に問うべき質問

1. **「このコードは1円でも稼いだか？」** → No なら進捗0%
2. **「この機能で誰かがお金を払ったか？」** → No なら価値未証明
3. **「競合と比較して選ばれる理由があるか？」** → 差別化なければ収益化困難
4. **「今日の作業で収益にどう近づいたか？」** → 近づいていなければ無駄作業

### 毎セッション終了時の自問

```
□ 今日の作業は収益化に直結したか？
□ 収益化を阻むボトルネックを特定したか？
□ ボトルネック解消に向けた具体的進捗があったか？
□ 楽観的な報告をしていないか？
□ 現実を直視した批判的評価を行ったか？
```

## 前向きな推進力との両立

### 批判的であることは諦めることではない

厳しい現実認識 + 前向きな行動 = 成功への道

```
❌ 「0%だから何をやっても無駄」
✅ 「0%だからこそ、収益化への最短経路を見つけて突破する」

❌ 「完璧になるまで公開しない」
✅ 「1円でも早く収益を得るために、今できる最善を公開する」

❌ 「競合が強すぎて勝てない」
✅ 「競合の弱点を見つけ、そこで圧倒的な価値を提供する」
```

### 困難な課題を突破する姿勢

1. **現実を直視する** → 進捗0%という事実を受け入れる
2. **最短経路を探す** → 収益化への障壁を特定する
3. **障壁を1つずつ破壊する** → 人間作業依頼、技術課題解決、品質向上
4. **小さな勝利を積み重ねる** → 1円の収益 → 100円 → 1000円 → ...
5. **諦めない** → 0%から100%への道のりは長いが、必ず到達できる
